# エンコーディングと制限

## サポートするエンコーディング

DocTree は標準ライブラリのみで実装されているため、サポートするのは以下のみです。

| 表示 | 種別 | 判定方法 |
|---|---|---|
| `UTF-8` | UTF-8 (BOMなし) | UTF-8 厳格デコードが成功すれば確定 |
| `UTF-8 (BOM)` | UTF-8 + BOM (`EF BB BF`) | BOM一致 |
| `UTF-16 LE` | UTF-16 リトルエンディアン + BOM (`FF FE`) | BOM一致 |
| `UTF-16 BE` | UTF-16 ビッグエンディアン + BOM (`FE FF`) | BOM一致 |

判定結果はステータスバーに表示されます。

## サポートしないエンコーディング

**Shift_JIS / EUC-JP / GB2312 / Big5 等の多バイトレガシエンコーディングはサポートしません。**

理由:
- 標準ライブラリ単体では `Encoding.GetEncoding("shift_jis")` を呼ぶために
  `System.Text.Encoding.CodePages` パッケージ (NuGet) が必要となり、本プロジェクトの
  「外部ライブラリ導入禁止」要件と矛盾するため。
- 自動判定 (chardet 等) も標準ライブラリでは現実的ではないため。

これらの非対応エンコーディングのファイルを開くと文字化けします。
**ソースを UTF-8 に変換するのが推奨される対応** です。

## 判定フロー

```
[読み込み] head = 先頭 binarySniffBytes バイト

if BOM (UTF-8 / UTF-16 LE / UTF-16 BE) → 確定
else if UTF-8 厳格デコードが成功 → UTF-8 (BOMなし)
else → settings の defaultEncoding にフォールバック (utf-8 / utf-16le / utf-16be)
```

## 行終端判定

最初に出現した CRLF / LF / CR で判定します。混在時は `Mixed` と表示。
TextBox での表示用には CRLF に正規化されます (元ファイルは変更されません)。

## 強制テキストオープン

拡張子未登録のファイルを開きたい場合、右クリック > **「テキストとして強制で開く」** で
バイナリ判定をスキップして開けます。エンコーディング判定は同じロジックで動きます。
